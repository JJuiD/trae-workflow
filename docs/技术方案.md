# 中间塔防 - 技术方案

## 1. 项目概述

### 1.1 技术栈选择
- **游戏框架**：原生 HTML5 Canvas + JavaScript（轻量级，无需构建工具）
- **网络方案**：PeerJS (WebRTC P2P，适用于1-4人局域网联机)
- **服务器**：无需后端

### 1.2 架构概览

## 5. 性能优化设计

### 5.1 敌人对象池

```javascript
class EnemyPool {
    constructor(maxSize) {
        this.pool = [];
        this.activeEnemies = [];
        for (let i = 0; i < maxSize; i++) {
            this.pool.push(this.createEnemy());
        }
    }

    spawn(enemyType) {
        const enemy = this.pool.length > 0
            ? this.pool.pop()
            : this.createEnemy(enemyType);
        enemy.init(enemyType);
        this.activeEnemies.push(enemy);
        return enemy;
    }

    despawn(enemy) {
        const idx = this.activeEnemies.indexOf(enemy);
        if (idx > -1) {
            this.activeEnemies.splice(idx, 1);
            this.pool.push(enemy);
        }
    }
}
```

### 5.2 视口裁剪

```javascript
class ViewportCulling {
    update(enemies, camera, viewport) {
        const visible = [];
        for (const enemy of enemies) {
            if (this.isInViewport(enemy, camera, viewport)) {
                visible.push(enemy);
            } else {
                enemy.pauseAI();
            }
        }
        return visible;
    }
}
```

### 5.3 分批渲染

```javascript
class BatchRenderer {
    render(enemies) {
        const byType = this.groupByType(enemies);
        for (const type in byType) {
            this.batchDraw(byType[type], type);
        }
    }
}
```

### 5.4 帧率保障策略

| 策略 | 说明 |
|------|------|
| 帧目标 | 稳定 60fps，最低可接受 30fps |
| 降级规则 | 敌人数量超过阈值时关闭粒子特效 |
| AI间隔 | 非视口敌人每3帧执行一次AI |
| 脏矩形渲染 | 只重绘变化区域 |

## 7. 数据流

### 7.1 游戏主循环

```
gameLoop(delta)
├── 1. WaveSystem.update()      - 波次时间检查、敌人生成
├── 2. EnemyMovement.update()   - 敌人位置更新、到达检测
├── 3. CombatSystem.update()    - 目标选择、攻击执行、伤害结算
├── 4. EconomySystem.update()   - 击杀奖励结算
├── 5. StateSync.broadcast()   - 游戏状态同步
└── 6. UI.refresh()            - UI显示更新
```

### 7.2 网络同步流程

```
玩家A操作 → 本地处理(验证金币、分配点位) → 序列化 → 广播 → 玩家B接收更新
```

## 8. 多人游戏流程

```
1. 房间创建（房主）
   └─> NetworkManager.start_host()
   └─> PeerManager.create_peer()
   └─> 等待玩家加入

2. 房间发现（玩家）
   └─> PeerManager.connect_to_host(host_id)
   └─> 收到完整游戏状态

3. 游戏开始
   └─> 所有玩家准备后，房主开始游戏
   └─> GameStateMachine.change_state(PLAYING)

4. 游戏中 - 同步位置、攻击、伤害，事件同步技能升级、击杀

5. 游戏结束 - 广播胜负结果，断开连接或返回房间
```

## 9. 关键算法

### 9.1 塔攻击目标选择

```javascript
selectTarget(tower, enemies) {
    const validEnemies = enemies.filter(
        e => e.position.distanceTo(tower.position) <= tower.attackRange
    );

    for (const enemy of validEnemies) {
        if (enemy.currentTarget === tower) {
            return enemy;
        }
    }

    validEnemies.sort((a, b) =>
        a.position.distanceTo(tower.position) - b.position.distanceTo(tower.position)
    );
    return validEnemies[0] || null;
}
```

### 9.2 敌人路径追踪

```javascript
updateMovement(enemy, delta) {
    const direction = this.targetPosition.clone().sub(enemy.position).normalize();
    enemy.position.add(direction.multiplyScalar(enemy.moveSpeed * delta));

    if (enemy.position.distanceTo(this.targetPosition) < 10) {
        this.emit('enemy_reached_base', enemy);
    }
}
```
