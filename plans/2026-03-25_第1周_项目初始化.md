# 第1周：项目初始化

## 任务目标

搭建开发环境、配置构建工具、创建基础目录结构，产出可运行的基础项目和Canvas画布。

## 任务拆解

1. **初始化npm项目**
   - 创建 `package.json`
   - 配置项目元数据（名称、版本、描述）
   - 配置构建脚本

2. **创建基础目录结构**
   - `src/` - 源代码目录
   - `assets/` - 游戏资源目录
   - `config/` - 配置文件目录
   - `test/` - 测试代码目录

3. **搭建Canvas画布**
   - 创建 `index.html` 入口文件
   - 初始化 Canvas 画布（800x600 或自适应）
   - 设置基础渲染循环

4. **配置开发工具**
   - 配置 npm 构建命令
   - 配置开发服务器（如需）

## 产出清单

- [ ] `package.json` - npm 项目配置
- [ ] `src/` 目录结构
- [ ] `index.html` - 游戏入口
- [ ] 基础 Canvas 渲染

## 验收标准

1. 项目可通过 `npm install` 安装依赖
2. 可通过 `npm run dev` 启动本地开发服务器
3. 浏览器可正常显示 Canvas 画布
4. 目录结构符合项目规范

## 技术方案

### 技术栈

- **游戏框架**：原生 HTML5 Canvas + JavaScript
- **网络方案**：PeerJS (WebRTC P2P)
- **构建工具**：npm + 原生 ES6 模块
