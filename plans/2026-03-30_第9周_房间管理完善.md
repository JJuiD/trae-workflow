# 第9周：房间管理完善

## 任务目标

完善房间管理功能，包括房间状态同步、玩家准备状态。

## 详细任务拆解

### 1. 房间状态同步

- [x] 实现房间游戏状态同步（等待中/游戏中）
- [x] 房主开始游戏后同步游戏状态给所有玩家
- [ ] 断线重连后恢复房间状态（滞后）

### 2. 玩家准备状态

- [x] 玩家准备/取消准备按钮
- [x] 显示玩家准备状态（已准备/未准备）
- [x] 房主显示开始游戏按钮（所有玩家准备后可用）
- [x] 同步准备状态给所有房间成员

## 产出清单

## 验收标准

### 房间状态同步
- [x] 房主开始游戏后，所有玩家进入游戏状态
- [ ] 断线重连可以恢复房间状态（滞后）

### 玩家准备状态
- [x] 玩家可以点击按钮切换准备/未准备状态
- [x] 所有玩家能看到其他玩家的准备状态
- [x] 所有玩家准备后，房主可开始游戏

## 技术方案

### 房间状态机
```
WAITING -> PLAYING -> (游戏结束) -> WAITING
```

### 准备状态同步
使用 PeerJS 广播玩家准备状态：
```javascript
{
  type: 'ready_changed',
  peerId: string,
  isReady: boolean
}
```

### 房间状态同步
```javascript
{
  type: 'room_state_changed',
  roomState: 'waiting' | 'playing'
}
```
