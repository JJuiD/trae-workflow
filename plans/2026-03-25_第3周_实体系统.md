# 第3周：实体系统

## 任务目标

设计并实现游戏实体系统，包括 Entity 基类、防御塔和敌人的抽象基类，以及基础属性管理。

## 详细任务拆解

### 3.1 Entity 基类
- [ ] 实体唯一标识（ID）
- [ ] 实体类型（type: 'tower' | 'enemy' | 'crystal'）
- [ ] 位置属性（x, y 像素坐标）
- [ ] 基础属性：
  - 生命值（hp, maxHp）
  - 魔法值（mp, maxMp）
  - 防御力（defense）：减伤比例
  - 攻击力（damage）
  - 暴击率（critRate）：暴击概率 0-1
  - 暴击伤害（critDamage）：暴击时的伤害倍率
  - 攻击速度（attackSpeed）：攻击间隔倍率，实际间隔 = attackCooldown / attackSpeed
  - 攻击间隔（attackCooldown）：基础攻击间隔
  - 当前冷却（currentCooldown）：当前剩余冷却时间
- [ ] 存活状态（isAlive）
- [ ] update(deltaTime) 方法框架
- [ ] render(ctx) 方法框架

### 3.2 防御塔基类 (Tower)
- [ ] 继承自 Entity
- [ ] 攻击范围（range）
- [ ] 目标敌人引用
- [ ] findTarget(enemies) 方法：选择最近敌人
- [ ] attack(target) 方法：攻击逻辑（包含暴击判定）
- [ ] 被动技能引用

### 3.3 敌人基类 (Enemy)
- [ ] 继承自 Entity
- [ ] 移动速度（speed）
- [ ] 初始目标：水晶
- [ ] **攻击逻辑**：
  - 优先朝水晶移动
  - 进入**塔的攻击范围**时，切换目标为塔
  - 摧毁塔后继续朝水晶移动
- [ ] 攻击水晶伤害（damageToCrystal）
- [ ] 死亡判定与移除

### 3.4 实体管理器 (EntityManager)
- [ ] 管理所有实体列表
- [ ] addEntity(entity)
- [ ] removeEntity(entityId)
- [ ] getEntitiesByType(type)
- [ ] updateAll(deltaTime)
- [ ] renderAll(ctx)

## 技术方案

### 实体类结构
```
Entity (基类)
├── id: string
├── type: string
├── x, y: number
├── hp, maxHp: number
├── mp, maxMp: number
├── defense: number (减伤比例)
├── damage: number
├── critRate: number (暴击率 0-1)
├── critDamage: number (暴击伤害倍率)
├── attackSpeed: number (攻击速度倍率)
├── attackCooldown: number (基础攻击间隔)
├── currentCooldown: number (当前冷却时间)
├── isAlive: boolean
├── update(deltaTime)
└── render(ctx)

Tower (继承 Entity)
├── range: number
├── passiveSkill: Skill
├── findTarget(enemies)
├── attack(target) - 含暴击判定
└── update(deltaTime)

Enemy (继承 Entity)
├── speed: number
├── damageToCrystal: number
├── update(deltaTime) - 移动逻辑
└── render(ctx)
```

### 移动逻辑（Enemy）
- 敌人拥有世界坐标（像素级）
- 初始目标为水晶，直线朝水晶移动
- **目标切换机制**：
  - 实时检测是否进入某塔的攻击范围
  - 进入塔攻击范围时，切换目标为该塔
  - 摧毁塔后，恢复原始目标（水晶）
- 每帧计算方向向量，标准化后乘以速度更新位置

### 性能优化方案（面向大量敌人）
- [ ] **对象池**：预创建敌人对象，复用而非频繁创建/销毁
- [ ] **空间分区**：网格划分地图，缩小目标检测范围
- [ ] **距离计算优化**：先做粗略检测（如曼哈顿距离），再精确计算
- [ ] **批量渲染**：敌人使用相同的渲染逻辑，减少 draw call

## 附录：友方单位系统（未来扩展）

### 核心需求
1. **单位类型抽象**：定义基础单位接口
   - 移动状态（可移动/不可移动）
   - 碰撞属性（碰撞单位/非碰撞单位）
   - 单位类型（防御塔/普通单位）

2. **单位创建机制**：工厂模式
   - 防御塔根据配置参数动态创建友方单位
   - 支持不同类型的单位生成

3. **状态管理**：统一接口
   - 生命值、攻击状态、移动状态
   - 生命周期管理（创建、存活、销毁）

4. **行为组件系统**：可扩展行为逻辑
   - 巡逻、驻守、追击等行为
   - 通过组合而非继承实现

5. **交互规则**
   - 碰撞单位：与敌人/其他单位发生物理交互
   - 非碰撞单位：穿透其他单位

### 架构设计
```
Unit (基类)
├── id, type, x, y, hp
├── isMovable, hasCollision
├── state: StateMachine
└── behaviors: Behavior[]

UnitFactory
├── createUnit(config): Unit
└── registerType(type, creator)

Behavior (接口)
├── PatrolBehavior
├── GuardBehavior
└── ChaseBehavior
```
